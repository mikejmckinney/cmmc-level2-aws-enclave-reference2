# NIST SP 800-171 r2 controls mapping

[`nist-800-171-mapping.csv`](./nist-800-171-mapping.csv) is the canonical
artifact: 110 rows, one per security requirement, mapping each control to
this repo's coverage.

## Source of truth

[NIST SP 800-171 Revision 2](https://csrc.nist.gov/pubs/sp/800/171/r2/upd1/final)
(Update 1, 21 Jan 2021). Control IDs and family codes come from that
document. Control titles in the CSV are **paraphrased**; we do not copy
NIST text verbatim.

## Tallies

| `addressed_by_repo` | Count |
|---|---|

| `partial` — Terraform helps; client must finish | **44** |
| `none` — Out of scope for the repo | **56** |

| Other counters | Count |
|---|---|
| `organizational_control = true` | 42 |
| `requires_client_config = true` | 86 |

The "none" / organizational counts are deliberately high: half of CMMC L2
is policy, training, IR planning, personnel screening, and physical
security — none of which a Terraform repo can deliver.

## Columns

| Column | Type | Description |
|---|---|---|
| `control_id` | string | NIST 800-171 r2 ID (e.g. `3.1.1`). Regex `^3\.\d{1,2}\.\d{1,2}$`. |
| `family` | string | Long family name. |
| `family_code` | string | Two-letter family code. One of `AC AT AU CM IA IR MA MP PS PE RA CA SC SI`. |
| `control_name` | string | Short title, paraphrased. |
| `description` | string | One-sentence paraphrase of the requirement. |
| `addressed_by_repo` | enum | `full` / `partial` / `none`. |
| `aws_services` | string | `;`-separated list of AWS services involved. |
| `terraform_resources` | string | `;`-separated list of `module.*` / `aws_*` references. |
| `requires_client_config` | bool-as-string | Client must complete technical work. |
| `organizational_control` | bool-as-string | Fundamentally a policy/process control. |
| `notes` | string | Caveats / references. Use `;` for in-cell separators (no commas). |

The JSON Schema in [`schema.json`](./schema.json) is the machine-readable
copy used by the CI check in [`.github/prompts/10-ci-and-compliance-checks.md`](../.github/prompts/10-ci-and-compliance-checks.md).

## How to update

1. Edit the `CONTROLS` list in [`scripts/gen-controls-csv.py`](../scripts/gen-controls-csv.py).
2. Re-run `python3 scripts/gen-controls-csv.py` from the repo root. The script asserts row count and uniqueness before writing.
3. If you change a `terraform_resources` value, confirm the resource still exists under `terraform/modules/` or `terraform/govcloud/`.
4. If you flip a row to `addressed_by_repo = full`, write or update the matching SSP statement in [`ssp/SSP.md`](../ssp/SSP.md).
5. Update the **Tallies** table above if counts shift.

## Verification

```bash
python3 scripts/gen-controls-csv.py
python3 - <<'PY'
import csv, re
rows = list(csv.DictReader(open("controls/nist-800-171-mapping.csv")))
assert len(rows) == 110
assert all(re.match(r"^3\.\d{1,2}\.\d{1,2}$", r["control_id"]) for r in rows)
print("OK")
PY
```
